# 云南省企业就业失业数据采集系统项目计划（最终版）

## 1. 项目概述
- 项目名称：云南省企业就业失业数据采集系统
- 项目目标：建设一个面向省、市、企业三级用户的就业失业数据采集与审核平台，实现企业备案、月度数据填报、分级审核、统计分析、系统维护与权限控制。
- 开发方式：基于 Agent 辅助编程方式完成需求分析、后端开发、前端开发、统计导出、系统维护和部署收尾。

## 2. 建设范围
- 用户与权限：省级、市级、企业三级用户登录与 RBAC 控制。
- 企业备案：企业基础信息填报、备案提交、备案审核。
- 月报填报：就业人数填报、减少类型与原因校验、月度上报时限控制。
- 审核流程：企业提交、市级审核、省级终审归档。
- 数据管理：按地市汇总统计、趋势分析、Excel 导出。
- 系统维护：上报时限设置、服务器监控、基础配置管理。
- 前端展示：Vue 3 页面、Element Plus 表单、ECharts 图表可视化。

## 3. 技术路线
- 后端：FastAPI + SQLAlchemy 2.0 + Pydantic v2 + JWT
- 数据库：PostgreSQL / MySQL 兼容设计，本地联调使用 SQLite
- 前端：Vue 3 + Vite + Element Plus + Axios + ECharts
- 迁移与部署：Alembic + Conda 环境 + 一键启动脚本

## 4. 主要里程碑
- 里程碑 1：完成需求梳理与数据模型设计
- 里程碑 2：完成后端核心接口与审核流转
- 里程碑 3：完成前端核心页面与动态表单
- 里程碑 4：完成省级统计、导出、系统维护模块
- 里程碑 5：完成联调、测试、部署与提交材料整理

## 6. 风险与应对
- 依赖安装风险：提前建立 Conda 环境并固定依赖版本。
- 数据库差异风险：模型设计保持 PostgreSQL / MySQL 基本兼容，本地使用 SQLite 快速验证。
- 需求变更风险：通过变更单记录新增模块和范围变化。
- 时间风险：优先完成主流程，收尾阶段集中处理部署、测试和文档。
- 演示风险：准备演示账号、一键启动脚本和固定演示路径。

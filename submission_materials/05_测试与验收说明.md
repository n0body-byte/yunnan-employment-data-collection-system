# 测试与验收说明

## 1. 测试目标
验证系统核心流程可运行，重点覆盖登录鉴权、企业备案、月报填报、三级审核、权限控制、统计导出、系统维护、查询筛选和前端构建。

## 2. 测试环境
- 操作系统：Windows
- Python 环境：Conda 环境 `yunnan-employment`
- 后端框架：FastAPI
- 前端框架：Vue 3 + Vite
- 联调数据库：SQLite（开发环境）
- 配置管理平台：GitHub

## 3. 主要测试项
### 3.1 后端校验测试
- 用户、企业、月报、通知等 Schema 校验
- 调查期人数小于建档期人数时的必填校验
- 密码哈希与 JWT 相关测试

### 3.2 接口流程测试
- 省级、市级、企业账号登录
- 企业备案提交与备案审核
- 月报提交
- 市级审核、退回、省级终审、上报部级
- 报表修订、导出、删除
- 上报时限配置与系统监控

### 3.3 权限控制测试
- 企业用户仅能查看本企业数据
- 市级用户仅能查看本地区数据
- 省级用户可查看全量数据
- 角色与权限分配后接口可按权限控制访问

### 3.4 前端页面测试
- 登录后按角色跳转到对应工作台
- 企业备案表单校验与行业联选可用
- 月报动态表单根据人数变化显示原因字段
- 省级管理页支持查询、查看、退回、导出
- 图表组件可正常构建并展示

### 3.5 文档与交付测试
- README 已改为中文并补齐运行说明
- 一键启动脚本存在
- 提交材料目录完整

## 4. 测试结果摘要
- Python 单元测试通过：`10` 项
- 后端导入验证通过：`import app.routes`
- 前端构建通过：`npm run build`
- 本地接口冒烟验证通过：登录、企业列表、RBAC、系统维护、导出等关键接口正常
- GitHub 可查看真实迭代提交记录

## 5. 验收结论
系统已经完成课程要求的核心功能，且进一步补齐了说明书中的省级管理、查询导出和提交材料同步内容，具备课堂演示与课程作业提交条件。

## 6. 当前限制说明
- 本地开发联调使用 SQLite，生产环境建议切换到 PostgreSQL 或 MySQL
- 对国家系统的数据交换目前实现的是本系统侧的导出与交换日志闭环，未对接真实外部平台协议
- PowerShell 终端可能出现中文显示乱码，但源码和文档文件本身为 UTF-8

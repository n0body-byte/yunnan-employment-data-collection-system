# 测试与验收说明

## 1. 测试目标
验证系统核心流程可运行，主要包括登录鉴权、企业备案、月报填报、审核流转、权限控制、统计导出和系统维护。

## 2. 测试环境
- 操作系统：Windows
- Python 环境：Conda 环境 yunnan-employment
- 后端框架：FastAPI
- 前端框架：Vue 3 + Vite
- 联调数据库：SQLite（开发环境）

## 3. 主要测试项
### 3.1 后端校验测试
- 用户、企业、月报、通知等 Schema 校验
- 月报中调查期人数小于建档期人数时的必填校验
- 登录密码哈希与 JWT 相关测试

### 3.2 接口流程测试
- 省级、市级、企业账号登录
- 企业备案提交
- 月报提交
- 市级审核 / 省级终审
- 统计汇总与企业导出
- 上报时限配置与监控接口

### 3.3 权限控制测试
- 企业用户仅能查看本企业数据
- 市级用户仅能查看本市数据
- 省级用户可查看全量数据

### 3.4 前端构建测试
- Vite 构建通过
- 路由可按角色跳转
- 动态表单可按人数变化展示原因字段
- 图表组件可接收接口数据并渲染

## 4. 测试结果摘要
- Python 单元测试通过：10 项
- 前端构建通过：npm run build 成功
- 本地接口冒烟验证通过：登录、企业列表、RBAC、系统维护接口正常
- 一键启动脚本已生成

## 5. 验收结论
系统已完成课程要求的核心功能，具备基本可运行性和展示条件，可用于课堂演示与课程作业提交。

## 6. 仍需说明的限制
- 本地开发联调使用 SQLite，生产环境建议切换到 PostgreSQL 或 MySQL
- 若需正式部署，需要根据机器实际 Conda 路径调整批处理脚本
- 项目计划迭代历史截图需使用真实 CM 工具截图补充

# 项目计划迭代历史截图说明

## 1. 目的
本文件用于说明“基于 CM 工具不断细化的项目计划迭代历史截图”应如何准备，以及建议截取哪些 GitHub 页面作为证据。

## 2. 项目计划迭代的核心证据
项目计划迭代主要体现在 GitHub 分支：
- `docs/project-plan-evolution`

对应版本标签：
- `plan-v1`
- `plan-v2`
- `plan-v3`

建议截图内容：
1. `docs/project-plan-evolution` 分支的提交历史
2. `plan-v1...plan-v2` 的 Compare 页面
3. `plan-v2...plan-v3` 的 Compare 页面

## 3. 三个计划版本的含义
- `版本1_初版`：项目目标、范围、一级任务和初步安排
- `版本2_细化版`：补充 WBS、里程碑、分工和风险
- `版本3_最终版`：补充测试、部署、提交材料和完整交付计划

## 4. 实施阶段迭代可选补充截图
除项目计划分支外，系统功能持续在 `main` 分支迭代。若老师希望查看“计划如何落地为实现”，可补充 `main` 分支近期提交截图，重点包括：
- `4d51ddb` 前端工作台与登录页重构
- `4260d7a` 省级综合管理页重构
- `1903f88` 路由懒加载与 Vendor 分包
- `cc81f37` Element Plus 与 ECharts 体积优化

## 5. 截图建议
- 每张截图尽量保留 GitHub 仓库名称、分支名、提交号和时间
- 如使用 Compare 页面，尽量让文件变更列表和提交标题同时出现在画面中
- 图下可以附一行图注，说明本次迭代新增了什么内容

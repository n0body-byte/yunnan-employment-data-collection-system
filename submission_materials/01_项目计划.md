# 云南省企业就业失业数据采集系统项目计划

## 1. 项目概述
- 项目名称：云南省企业就业失业数据采集系统
- 项目背景：根据《云南省企业就业失业数据采集系统工作说明书》完成课程项目开发与提交，形成可演示、可运行、可交付的软件系统。
- 项目目标：建设一个面向省级、市级、企业三级用户的就业失业数据采集与管理平台，实现企业备案、月报填报、分级审核、统计分析、系统维护、权限控制、导出与数据交换等业务。
- 开发方式：采用 Agent 辅助编程方式推进需求拆解、后端开发、前端开发、联调验证、文档完善和提交材料整理。
- 配置管理工具：Git + GitHub。

## 2. 建设范围
### 2.1 核心业务范围
- 用户登录、JWT 鉴权与角色权限控制
- 企业备案录入、备案上报、备案审核、备案查询、备案导出
- 月报填报、减少类型与原因校验、历史查询
- 企业提交、市级审核、省级终审、退回修改、上报部级
- 省级数据汇总、样本分析、对比分析、趋势分析
- Excel 导出、数据交换日志记录
- 上报时限管理、系统监控、用户管理、角色管理、通知管理

### 2.2 前端范围
- 登录页
- 企业工作台
- 市级工作台
- 省级统计总览页
- 省级综合管理页

### 2.3 交付范围
- 完整前后端源码
- 数据库迁移脚本
- 一键启动脚本
- 演示账号与初始化脚本
- README 与部署说明
- 项目计划、甘特图、变更单、测试与验收说明等课程材料

## 3. 技术路线
- 后端：FastAPI + SQLAlchemy 2.0 + Pydantic v2 + JWT
- 数据库：PostgreSQL / MySQL 兼容设计，本地联调使用 SQLite
- 前端：Vue 3 + Vite + Element Plus + Axios + ECharts
- 迁移：Alembic
- 运维：Conda 环境、批处理脚本、一键启动
- 配置管理：Git、GitHub 分支与标签

## 7. 风险与应对
- 依赖安装风险：提前建立 Conda 环境并固定依赖版本。
- 数据库兼容风险：模型设计保持 PostgreSQL / MySQL 基本兼容，本地使用 SQLite 快速联调。
- 需求扩展风险：使用变更单记录范围扩张，并按优先级补齐主流程。
- 前后端联调风险：优先稳定接口，再推进页面细节与导出逻辑。
- 演示风险：准备演示账号、一键启动脚本、固定演示路径和测试说明。
- 文档一致性风险：提交前统一同步 README、项目计划、变更单和验收说明。

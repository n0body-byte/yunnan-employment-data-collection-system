# 项目计划迭代实施说明

## 1. 目的
本说明用于生成“基于 CM 工具不断细化的项目计划迭代历史”。建议使用 Git 作为配置管理工具，对同一个项目计划文件进行三次递进式提交。

## 2. 已提供文件
- 07_项目计划_版本1_初版.md
- 07_项目计划_版本2_细化版.md
- 07_项目计划_版本3_最终版.md

## 3. 推荐操作步骤
### 第一步：提交初版计划
将 `07_项目计划_版本1_初版.md` 的内容覆盖到 `01_项目计划.md`，然后提交：

```powershell
git add submission_materials/01_项目计划.md
git commit -m "docs: 初版项目计划"
```

### 第二步：提交细化版计划
将 `07_项目计划_版本2_细化版.md` 的内容覆盖到 `01_项目计划.md`，然后提交：

```powershell
git add submission_materials/01_项目计划.md
git commit -m "docs: 细化项目计划，补充WBS和里程碑"
```

### 第三步：提交最终版计划
将 `07_项目计划_版本3_最终版.md` 的内容覆盖到 `01_项目计划.md`，然后提交：

```powershell
git add submission_materials/01_项目计划.md submission_materials/01_甘特图.md
git commit -m "docs: 完善项目计划，补充测试部署与甘特图"
```

截图要求：能清楚看到三次提交记录和提交说明。

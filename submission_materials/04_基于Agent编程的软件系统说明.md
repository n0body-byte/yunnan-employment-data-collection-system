# 基于 Agent 编程的软件系统说明

## 1. 系统名称
云南省企业就业失业数据采集系统

## 2. 系统概述
本系统面向省级、市级和企业三级用户，围绕企业备案、就业失业数据填报、审核流转、统计分析、系统维护和导出交换等业务建立完整的信息化支撑平台。系统按照课程要求实现了前后端分离结构，并具备演示、运行和提交条件。

## 3. 技术架构
- 后端：FastAPI + SQLAlchemy 2.0 + Pydantic v2 + JWT
- 前端：Vue 3 + Vite + Element Plus + Axios + ECharts
- 数据库：PostgreSQL / MySQL 兼容设计，本地开发使用 SQLite
- 迁移与交付：Alembic、Conda、一键启动脚本、README 中文文档
- 配置管理：Git + GitHub

## 4. 系统模块
### 4.1 企业端
- 企业备案录入与修改
- 备案状态查看
- 月报填报与动态校验
- 历史查询与详情查看
- 通知浏览
- 密码修改

### 4.2 市级端
- 本地区企业月报审核
- 退回并填写退回说明
- 通知发布与删除
- 密码修改

### 4.3 省级端
- 企业备案审核、查询、查看、导出
- 报表查看、退回修改、终审归档、上报部级
- 数据修订与修订日志保留
- 历史报表软删除
- 汇总统计、多维分析、图表分析
- 用户查询与导出
- 角色与权限管理
- 通知管理
- 上报时限设置
- 系统监控
- 国家系统数据交换日志

## 5. Agent 参与方式说明
本项目采用 Agent 辅助编程模式推进，具体体现在：
- 根据文字需求拆分模块并生成开发计划
- 生成模型、Schema、接口、权限依赖和迁移脚本
- 生成前端 Vue 组件、路由和交互逻辑
- 生成统计图表组件、导出函数、启动脚本和 README
- 在需求变更后快速补齐说明书缺口并同步更新提交材料

## 6. Agent 编程带来的效果
- 从需求到原型的实现速度明显提升
- 前后端与文档联动效率高
- 适合课程项目中频繁追加需求、持续收口的开发过程
- GitHub 能保留较清晰的功能迭代轨迹

## 8. 系统演示建议流程
1. 使用企业账号登录，填写并提交备案信息
2. 使用企业账号填报月报，演示人数下降时原因字段自动显现
3. 使用市级账号审核月报并退回/通过
4. 使用省级账号完成备案审核、终审归档、查看统计图表并导出
5. 进入系统维护与用户管理页面，演示上报时限、监控和查询导出
